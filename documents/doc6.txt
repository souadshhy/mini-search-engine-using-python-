A positional index extends the inverted index by storing where each term appears inside each document. This extra information is important for phrase queries. If a user searches for machine learning, the system should not return a document where machine appears at the start and learning appears much later in an unrelated sentence. The two words must appear next to each other and in the correct order. The positional index solves this by comparing term positions. It uses more memory than a simple inverted index, but it supports more precise searching and makes phrase retrieval possible.
