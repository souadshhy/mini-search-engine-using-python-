An inverted index is like the index at the end of a textbook, but built for a collection of documents. Instead of storing each document and searching through it line by line, the system stores each term with the documents where that term appears. For example, the term database may point to document two, document five, and document nine. This structure makes single term search and Boolean retrieval much faster. It also stores document frequency, which tells how many documents contain the term. Common words usually have high document frequency, while specific words often provide more useful evidence for retrieval.
