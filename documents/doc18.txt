Spelling correction improves the user experience when a query contains a small typing mistake. In a mini search engine, the correction can be based on the vocabulary created from the document collection. If the user writes machne, the system can compare it with known terms and suggest machine when the edit distance is small. This does not require a large language model. A simple dynamic programming edit distance algorithm is enough for a project demonstration. The important point is to show the original query, the corrected query, and the documents retrieved after correction.
